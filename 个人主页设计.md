
# 个人主页 — 核心需求文档（Solo Builder Edition）

## 1. 定位与调性

- **角色**：Solo Builder — 独立建设者，写代码、写文章、做产品。
- **语言**：中英双语，内容以中文为主，部分标题和标签附带英文。
- **受众**：fellow builders、潜在合作者、好奇的读者。
- **调性**：真实、松弛、技术人不装；不要企业味儿。

## 2. 页面结构（信息架构）

单页应用，所有内容在同一页面内，通过锚点导航切换。固定顶部导航栏（滚动后半透明毛玻璃效果）。包含以下6个模块（按顺序）：

| 模块 | 功能 |
|------|------|
| **Hero** | 首屏：一句话身份标识 + 动态状态行（当前正在做的项目/读的书） + CTA按钮（浏览作品、外链到X） |
| **Builds** | 主打作品展示：一个大卡片（主推项目）+ 2×2网格（其他精选项目） |
| **Projects** | 完整项目列表：表格形式显示所有项目（名称、描述、技术栈、状态、链接） |
| **Experiments** | 孵化中/实验性想法：纵向堆叠的卡片流（标题、标签、状态、备注） |
| **Writing** | 文章合集：时间线列表（标题、摘要、关键词、日期），点击进入详情页 |
| **About** | 关于我 + 联系方式：个人简介、技术哲学、当前学习/阅读、线下生活、社交链接 |

## 3. 交互与行为

- 导航支持锚点跳转（`#builds` 等），点击平滑滚动至对应 section。
- 页面内滚动，导航栏变为半透明毛玻璃效果。
- **响应式**：桌面默认多列布局；平板断点（1024px）和移动端断点（768px）需自适应，移动端导航变为汉堡菜单，卡片和表格变为单列。
- 卡片悬浮时应有轻微上移 + 阴影增强动效（轻量、不引入额外动画库）。
- 支持亮/暗色模式切换（通过 `data-theme` 属性控制，默认暗色）。
- 文章详情页是独立页面（路由 `/writing/:slug`）。

## 4. 内容数据需求

所有内容数据存储在 Obsidian vault 的 Markdown 文件中，通过 YAML front matter 定义，不直接写在 HTML 里。

### 数据源结构（在 vault 中）

```
主页内容/
├── published/          # 文章（writing）
├── projects/           # 项目（project）
└── experiments/        # 实验（experiment）
```

### 每个 Markdown 文件必须包含的字段（抽象）

| 类型 | 必含字段 |
|------|---------|
| **文章** | `title`, `slug`, `date`, `tags`, `summary`, `type: writing`, `published` |
| **项目** | `title`, `description`, `tech` (数组), `status` (released/development/concept), `links` (github/demo), `type: project` |
| **实验** | `title`, `tagline`, `status` (ideation/prototyping/paused), `notes`, `type: experiment` |

- 所有内容通过 **front matter** 标记 `published: false` 可下架。
- 排序：按日期倒序。

## 5. 自动化内容同步（流水线）

目标：在 Obsidian 写 Markdown → 自动同步到网站，无需手动编辑 JSON。

### 工作流程

1. 你向 GitHub 推送包含 vault 内容文件夹的仓库。
2. GitHub Actions 检测到 vault 文件变化（或手动触发）。
3. 运行一个同步脚本（Python 或 Node.js），实现：
   - 扫描指定 vault 文件夹，解析所有 Markdown 的 front matter。
   - 过滤 `published: true` 的内容。
   - 生成三个 JSON 数据文件：`content/writing.json`、`content/projects.json`、`content/experiments.json`。
   - 按日期倒序排序。
4. 如果数据有变化，自动 commit 并 push 到仓库。
5. 托管平台（Vercel/Netlify）自动检测变更并重新部署。

### 前置条件

- vault 与网站仓库在同一个 Git 仓库中（或使用 submodule）。
- GitHub Actions 运行环境无需本地依赖。

## 6. 技术栈与要求（抽象层）

- **工程实现**：单页应用，无框架依赖（纯 HTML + CSS + JavaScript），所有数据从 JSON 读取。
- **字体**：西文 Inter，中文 Noto Sans SC（通过 Google Fonts 加载）。
- **图标**：按需引入 Lucide 图标或自定义 SVG。
- **SEO**：预渲染 meta tags，支持 Open Graph / Twitter Card。
- **性能**：Lighthouse 得分 ≥ 90。
- **可访问性**：所有交互元素有 focus state，色彩对比度 ≥ 4.5:1。
- **部署**：Vercel 或 Netlify，自动从 GitHub 仓库部署。

## 7. 设计与开发的职责分工

| 角色 | 负责内容 |
|------|---------|
| **你（需求方）** | 提供以上需求文档 + Obsidian 内容源 |
| **专业设计AI** | 基于本需求，完成视觉方案（色彩、排版、动效细节、亮/暗色适配），输出设计稿或设计令牌 |
| **v0 / Bolt / Cursor** | 根据设计稿和本需求文档，实现一个可运行的静态站点，包含所有模块、响应式、自动化数据读取 |

## 8. 初始上线建议

优先实现 Hero + Builds + About 模块（核心），其次 Projects + Experiments + Writing 列表，最后加文章详情页和亮/暗色切换。自动化流水线可以与 MVP 同步搭建。
